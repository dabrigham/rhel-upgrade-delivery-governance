Technical Proof

This publishable bundle summarizes the sanitized governance demo for the RHEL upgrade delivery governance case study. It does not contain original enterprise code or production data.

What the Demo Proves

  • Controlled CSV inputs can be validated before downstream review.
  • Inventory records can be joined to change and task records deterministically.
  • Ready records can be separated from exception records with explicit reasons.
  • The workflow can be rerun and verified with synthetic data.

Local Commands

./scripts/run_demo.sh
./scripts/verify_public_export.sh

Expected Synthetic Behavior

  • Deterministic classification of ready and exception records.
  • Explicit review reasons for held records.
  • Safe separation of wave-review records from human-review exceptions.

Publication Boundary

Public release still requires human review and explicit approval. This page is safe static proof, not a claim of production upgrade completion.
